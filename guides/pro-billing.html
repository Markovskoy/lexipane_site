Pro access and AI credits

ProSecurityTelegram keys · monthly allowance · top-ups

Pro keys must be verified by the Lexipane subscription backend. The desktop app should only store a signed result or license token; it should not decide by itself that an arbitrary key is valid.

Safe architecture

  • Telegram bot: accepts payment, creates a subscription record, returns a license key.
  • License API: validates the key, returns plan, expiry date and monthly AI credit limit.
  • AI proxy: holds the real OpenAI key server-side, tracks ruble spend by model and blocks requests after the monthly allowance.
  • Top-ups: one-time codes add a fixed balance and are redeemed once on the server.

Credit policy

The planned default is a monthly allowance, for example 200 RUB of AI usage, with top-up packs such as 50 RUB, 100 RUB and 150 RUB. Exact token counts depend on the selected model and current OpenAI pricing, so the backend should calculate cost at request time.

What the app should never do

  • Never ship the master OpenAI key inside the desktop app.
  • Never create OpenAI API keys from the client.
  • Never unlock Pro just because a key matches a local pattern.
  • Never trust client-side credit counters for paid cloud usage.

Pro-доступ и AI-кредиты

ProSecurityTelegram keys · месячный лимит · пополнения

Pro-ключи должны проверяться backend подписок Lexipane. Desktop-приложение должно хранить только подписанный результат или license token; оно не должно само решать, что произвольный key валиден.

Безопасная архитектура

  • Telegram bot: принимает оплату, создает запись подписки, возвращает license key.
  • License API: проверяет key, возвращает plan, дату окончания и месячный лимит AI-кредитов.
  • AI proxy: хранит реальный OpenAI key на сервере, считает расход в рублях по модели и блокирует запросы после месячного лимита.
  • Top-ups: одноразовые коды добавляют фиксированный баланс и погашаются один раз на сервере.

Политика кредитов

Планируемый default - месячный лимит, например 200 RUB на AI usage, плюс top-up packs на 50 RUB, 100 RUB и 150 RUB. Точное число tokens зависит от модели и актуальных цен OpenAI, поэтому backend должен считать стоимость на момент запроса.

Что приложение не должно делать

  • Не вшивать master OpenAI key в desktop-приложение.
  • Не создавать OpenAI API keys из клиента.
  • Не открывать Pro только потому, что key совпал с локальным шаблоном.
  • Не доверять клиентским счетчикам кредитов для paid cloud usage.
