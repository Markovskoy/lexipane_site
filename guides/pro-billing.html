Pro access and AI credits

Pro Security Telegram keys · monthly allowance · top-ups

Pro keys must be verified by the Lexipane subscription backend. The desktop app should only store a signed result or license token; it should not decide by itself that an arbitrary key is valid.

Safe architecture

  • Telegram bot: accepts payment, creates a subscription record, returns a license key.
  • License API: validates the key, returns plan, expiry date and monthly AI credit limit.
  • AI proxy: holds the real OpenAI key server-side, tracks ruble spend by model and blocks requests after the monthly allowance.
  • Top-ups: one-time codes add a fixed balance and are redeemed once on the server.

Credit policy

The planned default is a monthly allowance, for example 200 ₽ of AI usage, with top-up packs such as 50 ₽, 100 ₽ and 150 ₽. Exact token counts depend on the selected model and current OpenAI pricing, so the backend should calculate cost at request time.

What the app should never do

  • Never ship the master OpenAI key inside the desktop app.
  • Never create OpenAI API keys from the client.
  • Never unlock Pro just because a key matches a local pattern.
  • Never trust client-side credit counters for paid cloud usage.
