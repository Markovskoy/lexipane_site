Custom LLM endpoints

AdvancedOpenAI-compatibleLocal server · base URL · validation

Custom endpoints are for users who already run their own model gateway, local server, or OpenAI-compatible API. This mode is separate from Lexipane Cloud billing: requests go from your computer to the endpoint you configure.

What to enter

  • Base URL: an HTTPS OpenAI-compatible base URL such as https://example.com/v1, or a local URL such as http://127.0.0.1:11434/v1.
  • API key: the token required by your endpoint. Local servers may allow an empty or dummy value.
  • Model name: the exact model id exposed by your server.

Validation before saving

Lexipane should test the endpoint with a small request before saving it. If validation fails, the settings screen should explain whether the URL is unreachable, the key is rejected, the model is missing, or the response is not OpenAI-compatible.

Privacy boundary

In custom endpoint mode, captured text is sent only to the endpoint you configured. It does not use Lexipane Cloud credits and does not go through the Lexipane backend unless you deliberately enter a Lexipane backend URL yourself.

Good local candidates

  • Local OpenAI-compatible gateways.
  • Self-hosted model servers on your LAN.
  • Company proxy endpoints that expose chat completions with OpenAI-compatible request and response shapes.

Свои LLM endpoints

AdvancedOpenAI-compatibleЛокальный сервер · base URL · проверка

Custom endpoints нужны пользователям, у которых уже есть свой model gateway, локальный сервер или OpenAI-compatible API. Этот режим отделен от биллинга Lexipane Cloud: запросы идут с вашего компьютера на endpoint, который вы указали.

Что указывать

  • Base URL: HTTPS OpenAI-compatible base URL вроде https://example.com/v1 или локальный URL вроде http://127.0.0.1:11434/v1.
  • API key: токен, который требует ваш endpoint. Локальные серверы иногда допускают пустое или dummy-значение.
  • Model name: точный id модели, который отдает ваш сервер.

Проверка перед сохранением

Lexipane должен отправлять маленький тестовый запрос до сохранения настроек. Если проверка не прошла, экран настроек должен объяснить причину: URL недоступен, key отклонен, модель не найдена или ответ не похож на OpenAI-compatible API.

Граница приватности

В custom endpoint режиме захваченный текст отправляется только на endpoint, который вы настроили. Он не тратит кредиты Lexipane Cloud и не идет через backend Lexipane, если вы сами не указали backend Lexipane как URL.

Хорошие локальные варианты

  • Локальные OpenAI-compatible gateways.
  • Self-hosted model servers в вашей локальной сети.
  • Корпоративные proxy endpoints, которые поддерживают chat completions в OpenAI-compatible формате.
